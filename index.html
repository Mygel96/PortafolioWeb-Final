Importancia de un Ingeniero de Sistemas

Los ingenieros de sistemas diseñamos y construimos soluciones tecnológicas que automatizan procesos, permiten la toma de decisiones basada en datos y mejoran la experiencia de usuarios y clientes. Responsabilidades y valores clave:

  • Resolver problemas reales mediante software confiable y mantenible.
  • Priorizar la experiencia del usuario, rendimiento y seguridad.
  • Trabajar colaborativamente con equipos multidisciplinares (diseño, producto, datos).
  • Adoptar buenas prácticas y estándares para facilitar escalabilidad y mantenibilidad.

Proyectos destacados

Proyecto: Sistema Web Inteligente para Clasificación de Patentes (PyPatentAlice)

Se desarrolló de forma grupal y para el parcial final del curso, una aplicación web inteligente full stack orientada al análisis automático de textos legales de patentes, basada en Inteligencia Artificial y Procesamiento de Lenguaje Natural (NLP). El sistema clasifica reivindicaciones de patentes según su posible no patentabilidad bajo el criterio de la decisión Alice v. CLS Bank.

El backend fue implementado en Python (Flask) bajo una arquitectura cliente–servidor en capas, integrando un modelo de Machine Learning supervisado (SVM + TF-IDF) para clasificación binaria de texto. Se incorporó autenticación segura con JWT, persistencia en MySQL y una API REST para análisis y gestión de usuarios.

El frontend fue desarrollado en React, siguiendo una arquitectura basada en componentes, con una interfaz limpia, responsive y orientada a la experiencia de usuario. Permite el ingreso de texto, visualización de resultados, manejo de sesiones y consulta de historiales de análisis.

Este proyecto consolida mis conocimientos en desarrollo web backend y frontend, seguridad, arquitectura de software e integración de modelos de IA en aplicaciones reales, aplicando buenas prácticas de escalabilidad, mantenibilidad y ética en sistemas inteligentes.

Reflexión

Durante este curso he ido construyendo una base práctica y reflexiva: el cuaderno de aprendizaje contiene temas, descripciones y mis propias definiciones, procedimientos y detalles de trabajo que me ayudan a consolidar lo aprendido. Mi objetivo es combinar conocimientos técnicos con buenas prácticas de diseño para crear interfaces útiles, accesibles y mantenibles. Los compromisos que guían mi aprendizaje incluyen: practicar la semántica y accesibilidad en cada ejercicio; estructurar estilos y componentes de forma coherente; aplicar control de versiones en todos los proyectos; y documentar cada entrega con reflexiones y enlaces relevantes.

A futuro planeo integrar pequeñas pruebas automatizadas, persistir los recursos del cuaderno (por ejemplo usando localStorage o un repositorio) y construir un proyecto final que reúna HTML, CSS, JavaScript/React y buenas prácticas de accesibilidad.
